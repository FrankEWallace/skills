# Software Development Skills & Standards

> A comprehensive guide for AI agents and developers on creating beautiful, maintainable, professional software systems.

**Philosophy**: Code is read 10x more than it's written. Prioritize clarity, elegance, and user experience over quick hacks.

## How to Use This Guide

### For Developers
1. Start with [Core Principles](./core-principles.md) to understand our philosophy
2. Reference specific guides as needed for your work
3. Contribute improvements via PR

### For AI Agents (Claude, Copilot, etc.)
When working on a project:
1. **Always** read [Core Principles](./core-principles.md) and [AI Agent Instructions](./ai-agent-instructions.md) first
2. Load relevant specialized guides based on the task:
   - Frontend work? Read [Frontend Development](./frontend.md) and [UI/UX Design](./ui-ux-design.md)
   - Backend work? Read [Backend Development](./backend.md) and [Security](./security.md)
   - Fixing bugs? Check [Anti-Patterns](./anti-patterns.md)
   - Writing tests? Read [Testing](./testing.md)
3. Follow the patterns and avoid the anti-patterns

## Contributing

This guide evolves with our practices. To contribute:

1. Open a PR with your changes
2. Provide real-world examples
3. Explain the "why" behind the rule
4. Link to external resources when helpful
